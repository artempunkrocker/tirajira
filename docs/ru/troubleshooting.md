# Решение частых проблем

## "Ошибка при создании задачи: You must specify a summary of the issue"

**Причина:** Отсутствует обязательное поле `summary` в задаче.

**Решение:** Убедитесь, что каждая задача имеет поле `summary` с непустым значением:

```json
{
  "project": {"key": "PROJ"},
  "summary": "Название задачи",  // ← Обязательное поле
  "issuetype": {"name": "Task"}
}
```

## "Ошибка при создании задачи: Project key is required"

**Причина:** Отсутствует или некорректно указан ключ проекта.

**Решение:** Проверьте, что у каждой задачи есть поле `project.key` с существующим ключом проекта в Jira:

```json
{
  "project": {"key": "PROJ"},  // ← Проверьте ключ проекта в Jira
  "summary": "Название задачи",
  "issuetype": {"name": "Task"}
}
```

## "Ошибка при создании задачи: Issue type is required"

**Причина:** Отсутствует или некорректно указан тип задачи.

**Решение:** Убедитесь, что у каждой задачи указан тип задачи в поле `issuetype.name`:

```json
{
  "project": {"key": "PROJ"},
  "summary": "Название задачи",
  "issuetype": {"name": "Task"}  // ← Может быть Task, Bug, Story и т.д.
}
```

## "Ошибка аутентификации"

**Причина:** Некорректные параметры подключения к Jira.

**Решение:**
1. Проверьте правильность URL сервера Jira
2. Убедитесь, что email и API токен действительны
3. Для Jira Server проверьте действительность PAT токена

```env
# Для Jira Cloud
JIRA_SERVER=https://your-company.atlassian.net  # ← Проверьте URL
JIRA_EMAIL=your-email@company.com               # ← Проверьте email
JIRA_API_TOKEN=your-valid-api-token             # ← Проверьте токен

# Для Jira Server
JIRA_SERVER=https://your-jira-server.company.com # ← Проверьте URL
JIRA_PAT_TOKEN=your-valid-pat-token             # ← Проверьте токен
```

## "Файл не найден"

**Причина:** Указан неверный путь к файлу с задачами.

**Решение:** Проверьте правильность пути к файлу:

```bash
# Проверьте, что файл существует
ls -la tasks.json

# Используйте абсолютный или относительный путь
tirajira import ./tasks.json
tirajira import /full/path/to/tasks.json
```

## "Путь вне допустимой директории"

**Причина:** Попытка доступа к файлу вне текущей рабочей директории (защита от directory traversal атак).

**Решение:** Убедитесь, что файл с задачами находится в текущей рабочей директории или её поддиректориях:

```bash
# Проверьте текущую директорию
pwd

# Убедитесь, что файл находится в текущей директории или её поддиректориях
ls -la ./path/to/tasks.json

# Используйте относительный путь к файлу
tirajira import ./tasks.json
tirajira import ./subdirectory/tasks.json
```

TiraJira реализует защиту от атак directory traversal, блокируя доступ к файлам вне текущей рабочей директории. Это предотвращает возможность получения доступа к системным файлам или другим файлам вне рабочей директории.

## "Connection refused" или "Timeout"

**Причина:** Проблемы с сетевым подключением к Jira.

**Решение:**
1. Проверьте доступность Jira сервера в браузере
2. Убедитесь, что нет проблем с сетью или VPN
3. Проверьте настройки брандмауэра

## "Custom field not found"

**Причина:** Указан несуществующий ID кастомного поля.

**Решение:**
1. Проверьте ID кастомных полей в настройках Jira
2. Убедитесь, что используете правильный формат:

```json
{
  "project": {"key": "PROJ"},
  "summary": "Задача с кастомным полем",
  "issuetype": {"name": "Task"},
  "customfield_10001": "Значение поля"  // ← Проверьте ID поля
}
```

## "Insufficient permissions"

**Причина:** У пользователя недостаточно прав для создания задач в проекте.

**Решение:**
1. Проверьте права пользователя в Jira
2. Убедитесь, что пользователь может создавать задачи в указанном проекте
3. Проверьте, что тип задачи разрешен в проекте

## "API rate limit exceeded"

**Причина:** Превышены лимиты API Jira.

**Решение:**
Используйте новые параметры управления ограничением частоты запросов:

1. Уменьшите количество одновременных запросов:
   ```bash
   tirajira import tasks.json --max-concurrent-requests 3
   ```

2. Увеличьте интервал между запросами:
   ```bash
   tirajira import tasks.json --min-request-interval 15.0
   ```

3. Используйте оба параметра вместе для более консервативного подхода:
   ```bash
   tirajira import tasks.json --max-concurrent-requests 3 --min-request-interval 15.0
   ```

Более подробную информацию см. в [документации по ограничению частоты запросов](rate_limiting.md).

## Как включить подробное логирование

Для диагностики проблем включите подробное логирование:

```bash
tirajira import tasks.json --verbose
```

Это поможет понять, на каком этапе происходит ошибка.
