# Управление ограничением частоты запросов

TiraJira использует механизм ограничения частоты запросов для предотвращения превышения лимитов API Jira. Это особенно важно при массовом создании задач, чтобы избежать блокировки вашего аккаунта или IP-адреса со стороны сервера Jira.

## Подход к ограничению частоты запросов

TiraJira реализует модульную архитектуру для управления ограничением частоты запросов, состоящую из четырех основных компонентов:

1. **RateLimitController** - управляет тайминговыми задержками между запросами
2. **TaskProcessor** - обрабатывает отдельные задачи Jira
3. **ConcurrencyManager** - управляет параллельной обработкой задач
4. **JiraClientInterface** - интерфейс для взаимодействия с Jira API

Эта архитектура обеспечивает четкое разделение ответственностей между компонентами, что упрощает тестирование, расширение и поддержку системы ограничения частоты запросов.

## Компоненты системы ограничения частоты запросов

### RateLimitController

**Ответственность:** Управление минимальным интервалом между запросами к Jira API.

Этот компонент обеспечивает соблюдение минимального временного интервала между последовательными запросами к Jira API. Он использует потокобезопасную блокировку для координации доступа к времени последнего запроса и при необходимости выполняет ожидание перед отправкой следующего запроса.

**Пример использования:**
```python
from tirajira.rate_limit_controller import RateLimitController

controller = RateLimitController(min_request_interval=2.0)
controller.wait_if_needed()  # Ждет, если необходимо, чтобы соблюсти интервал
```

### TaskProcessor

**Ответственность:** Обработка отдельных задач Jira с учетом ограничений частоты запросов.

Этот компонент отвечает за создание задач в Jira API и привязку их к эпикам при необходимости. Он взаимодействует с RateLimitController для соблюдения временных ограничений между запросами и предоставляет подробную информацию о результатах обработки для формирования отчетов.

**Пример использования:**
```python
from tirajira.task_processor import TaskProcessor
from tirajira.rate_limit_controller import RateLimitController

processor = TaskProcessor(jira_client)
controller = RateLimitController(min_request_interval=1.0)

task_detail, success = processor.process_single_issue(
    issue_data, 0, controller
)
```

### ConcurrencyManager

**Ответственность:** Управление параллельной обработкой задач с различными стратегиями обработки ошибок.

Этот компонент управляет параллельной обработкой задач с учетом максимального количества одновременных запросов. Он поддерживает две стратегии обработки ошибок:
- Остановка при первой ошибке (`stop_on_error=True`)
- Продолжение обработки несмотря на ошибки (`stop_on_error=False`)

**Пример использования:**
```python
from tirajira.concurrency_manager import ConcurrencyManager

manager = ConcurrencyManager(
    max_concurrent_requests=5,
    verbose=True
)

successful_count, processing_details = manager.process(
    issues=issues_list,
    processor_func=process_func,
    stop_on_error=False
)
```

### JiraClientInterface

**Ответственность:** Абстрактный интерфейс для взаимодействия с Jira API.

Этот интерфейс позволяет создавать задачи и привязывать их к эпикам. Он используется всеми другими компонентами для взаимодействия с Jira API, что упрощает тестирование и позволяет легко заменить реализацию.

## Основные параметры

### max_concurrent_requests

Определяет максимальное количество одновременных запросов к Jira API.

- **Назначение**: Контроль параллелизма при создании задач
- **Тип**: Целое число
- **По умолчанию**: 10

### min_request_interval

Задает минимальный интервал между последовательными запросами к Jira API в секундах.

- **Назначение**: Предотвращение превышения частоты запросов
- **Тип**: Число с плавающей точкой (секунды)
- **По умолчанию**: 1.0

## Значения по умолчанию и их обоснование

Консервативные значения по умолчанию выбраны для минимизации риска превышения лимитов API Jira:

- `max_concurrent_requests: 10` - умеренный уровень параллелизма, позволяющий эффективно создавать задачи без чрезмерной нагрузки на сервер
- `min_request_interval: 1.0` - безопасный интервал в 1 секунду между запросами для соответствия большинству стандартных лимитов Jira API

Эти значения обеспечивают надежную работу в большинстве конфигураций Jira без необходимости дополнительной настройки.

## Примеры использования

### Базовое использование с параметрами по умолчанию

```bash
tirajira import tasks.json
```

### Уменьшение параллелизма для чувствительных систем

```bash
tirajira import tasks.json --max-concurrent-requests 3
```

### Увеличение интервала между запросами

```bash
tirajira import tasks.json --min-request-interval 5.0
```

### Комбинированная настройка для крупных импортов

```bash
tirajira import large_tasks.json --max-concurrent-requests 5 --min-request-interval 2.0
```

### Настройка при возобновлении выполнения

```bash
tirajira resume report.json --max-concurrent-requests 3 --min-request-interval 5.0
```

## Рекомендации по настройке

### Для Jira Cloud

- Начните с параметров по умолчанию
- При возникновении ошибок "API rate limit exceeded" уменьшите `max_concurrent_requests` или увеличьте `min_request_interval`
- Для больших импортов рекомендуется использовать более высокие интервалы

### Для Jira Server/Data Center

- Можно использовать более агрессивные настройки
- Проверьте внутренние политики ограничения частоты вашего сервера
- Поэкспериментируйте с увеличением `max_concurrent_requests` для повышения производительности

### Общие рекомендации

1. **Начинайте с значений по умолчанию** - они протестированы и безопасны
2. **Изменяйте один параметр за раз** - это поможет определить оптимальные настройки
3. **Используйте подробный режим (`--verbose`) для мониторинга** - вы сможете видеть, как параметры влияют на производительность
4. **Документируйте успешные конфигурации** - сохраняйте команды, которые хорошо работают в вашей среде

## Устранение неполадок

### "API rate limit exceeded"

Эта ошибка указывает на превышение лимитов API Jira. Решения:

1. Уменьшите количество одновременных запросов:
   ```bash
   tirajira import tasks.json --max-concurrent-requests 3
   ```

2. Увеличьте интервал между запросами:
   ```bash
   tirajira import tasks.json --min-request-interval 5.0
   ```

3. Используйте комбинацию обоих подходов:
   ```bash
   tirajira import tasks.json --max-concurrent-requests 3 --min-request-interval 5.0
   ```

### Медленное выполнение

Если импорт происходит слишком медленно:

1. Убедитесь, что вы не используете избыточно консервативные настройки
2. Постепенно увеличивайте `max_concurrent_requests` (до 10-15)
3. Уменьшите `min_request_interval` (но не менее 0.5 секунд)
4. Мониторьте появление ошибок при изменении параметров

### Рекомендации по мониторингу

- Используйте параметр `--verbose` для просмотра подробного лога
- Обратите внимание на сообщения об ошибках API
- При большом количестве задач рассмотрите возможность использования отчетов для возобновления выполнения после ошибок

## Программное использование

Помимо командной строки, систему ограничения частоты запросов можно использовать программно:

```python
from tirajira.rate_limiter import RateLimiter
from tirajira.jira_client import JiraClient

jira_client = JiraClient(...)  # Настройте подключение к Jira
rate_limiter = RateLimiter(
    jira_client=jira_client,
    max_concurrent_requests=5,
    min_request_interval=2.0
)

issues = [
    {"project": {"key": "PROJ"}, "summary": "Задача 1", "issuetype": {"name": "Task"}},
    {"project": {"key": "PROJ"}, "summary": "Задача 2", "issuetype": {"name": "Task"}}
]

successful_count, processing_details = rate_limiter.process(issues)
```
