# Часто задаваемые вопросы

## Можно ли использовать TiraJira с Jira Server и Jira Cloud?

Да, TiraJira поддерживает оба типа Jira:
- **Jira Cloud** - через Basic Auth (email + API token)
- **Jira Server/Data Center** - через Personal Access Token

## Какие форматы файлов поддерживаются?

Поддерживаются 5 форматов:
- JSON (рекомендуется для сложных структур)
- YAML (удобочитаемый формат)
- CSV (простые таблицы)
- Excel (XLSX, для пользователей Office)
- XML (для интеграций)

## Как узнать ID кастомных полей в Jira?

1. Перейдите в "Jira Administration" → "Issues" → "Custom Fields"
2. Найдите нужное поле и наведите на него мышь
3. ID будет отображен в URL или всплывающей подсказке
4. Или используйте Jira REST API: `GET /rest/api/2/field`

## Можно ли создавать подзадачи?

Да, укажите родительскую задачу через поле `parent.key`:

```json
{
  "project": {"key": "PROJ"},
  "summary": "Подзадача",
  "issuetype": {"name": "Sub-task"},
  "parent": {"key": "PROJ-123"}
}
```

## Как работает привязка к эпику?

После создания задачи TiraJira автоматически связывает её с эпиком, если указано поле `epic_key`:

```json
{
  "project": {"key": "PROJ"},
  "summary": "Задача в эпике",
  "issuetype": {"name": "Task"},
  "epic_key": "PROJ-100"  // ← Будет автоматически привязана к эпику
}
```

## Можно ли обновлять существующие задачи?

В текущей версии TiraJira поддерживает только создание новых задач. Для обновления существующих задач используйте другие интрументы или Jira REST API напрямую.

## Как управлять ограничением частоты запросов?

По умолчанию используется ограничение на 5 одновременных запросов с интервалом 10 секунд. Измените параметры ограничения частоты запросов:

```bash
# Уменьшение количества одновременных запросов
tirajira import tasks.json --max-concurrent-requests 3

# Увеличение интервала между запросами
tirajira import tasks.json --min-request-interval 15.0
```

⚠️ Слишком агрессивные настройки могут привести к ошибкам API при превышении лимитов Jira.

## Как отладить проблемы с созданием задач?

1. Включите подробное логирование:
   ```bash
   tirajira import tasks.json --verbose
   ```

2. Проверьте формат файла на соответствие примерам

3. Убедитесь, что все обязательные поля заполнены

4. Проверьте права доступа к проекту в Jira

## Можно ли использовать TiraJira в автоматизированных скриптах?

Да, TiraJira отлично подходит для автоматизации. Вы можете:
- Интегрировать её в CI/CD пайплайны
- Использовать в bash/python скриптах
- Запускать по расписанию через cron
- Интегрировать с другими системами через отчеты

## Как обрабатываются ошибки при создании задач?

TiraJira по умолчанию продолжает обработку даже при ошибках отдельных задач. Вы можете:
- Использовать `--stop-on-error` для остановки при первой ошибке
- Просматривать отчеты для анализа ошибок
- Использовать команду `resume` для повторной обработки неудачных задач
- Использовать команду `extract-failed` для извлечения неудачных задач в отдельный файл

## Где хранятся отчеты по умолчанию?

Если имя файла отчета не указано явно, то используется формат JSON с автоматически сгенерированным именем файла вида `tirajira_report_YYYYMMDD_HHMMSS.json`.

## Можно ли изменить формат отчета?

Да, формат отчета определяется по расширению указанного файла:
- `.json` - JSON формат
- `.csv` - CSV формат
- `.xlsx` - Excel формат
- `.yaml` - YAML формат
- `.xml` - XML формат

## Какие права нужны пользователю Jira для работы с TiraJira?

Пользователь должен иметь права на:
- Создание задач в указанных проектах
- Просмотр и редактирование используемых полей
- Создание связей с эпиками (если используется `epic_key`)
- Создание подзадач (если создаются подзадачи)
